# 🎯 바이브코딩 초보자 가이드: AI와 함께하는 11단계 앱 만들기

환영합니다! 이 매뉴얼은 코딩을 전혀 모르는 초보자도 AI의 도움을 받아 아이디어를 실제 작동하는 웹 앱으로 완성할 수 있도록 돕는 **'바이브코딩 11단계 워크플로우'** 사용자 가이드입니다. 

---

## 🛠 시작하기 전에: 기본 세팅

바이브코딩 앱을 제대로 사용하기 위해 딱 한 가지 준비가 필요합니다.
1. 우측 상단의 **[⚙️ API 설정]** 버튼을 클릭합니다.
2. 발급받은 **OpenAI API Key**를 입력하고 저장합니다. (AI가 프롬프트를 처리하기 위해 필요합니다.)
3. 만약 어떤 식으로 작동하는지 먼저 보고 싶다면, 우측 상단의 **[📝 데모 모드]**를 클릭해 예시 시나리오('할일 관리 앱')를 불러와 전체 흐름을 살펴볼 수 있습니다.

> **💡 핵심 꿀팁 (프롬프트 릴레이)**
> 이 앱은 **[현재 단계의 결과물]**을 **[다음 단계의 입력값]**으로 넘기며 눈덩이처럼 앱을 완성해가는 구조입니다.
> AI가 답변을 완료하면 항상 하단의 **[➡️ 다음 단계 프롬프트로 보내기]** 버튼을 꼭 눌러주세요!

---

## 🚀 단계별 가이드

### 💡 1단계: 아이디어 (Idea)
- **목표:** "무엇을 만들 것인가?"를 명확히 합니다.
- **실행 방법:** 화면의 [💬 AI 프롬프트 템플릿] 영역에 자신이 만들고 싶은 앱의 대상(타겟)과 해결할 문제를 괄호`[ ]` 안에 적습니다.
- **실행:** **[🚀 직접 시행]** 버튼을 누르면 AI가 아이디어를 구체화해 줍니다.
- **마무리:** 결과물을 확인하고 **[다음 단계 프롬프트로 보내기]**를 클릭합니다.

### 📄 2단계: PRD.md (요구사항 정의)
- **목표:** 아이디어를 구체적인 기능 목록(MVP)으로 정리합니다.
- **실행 방법:** 1단계에서 넘어온 결과물을 바탕으로 AI가 자동으로 PRD(제품 요구사항 문서) 프롬프트 초안을 만들어 놓았습니다. 바로 **[🚀 직접 시행]**을 누르세요.
- **결과물:** 핵심 기능 3가지와 성공 지표가 담긴 문서가 생성됩니다.

### 🔄 3단계: USER_FLOW.md (사용자 흐름)
- **목표:** 사용자가 앱에 들어와서 목표를 달성하기까지의 클릭 순서를 정의합니다.
- **실행 방법:** 넘겨받은 PRD를 바탕으로 앱의 화면 이동 경로를 정리합니다. **[🚀 직접 시행]**을 클릭하세요.

### 🖥️ 4단계: UI_SPEC.md (화면 명세서)
- **목표:** 각 화면에 어떤 버튼이나 입력창이 들어가야 할지 상세 목록을 만듭니다.
- **실행 방법:** 자동으로 작성된 프롬프트를 **[🚀 직접 시행]**하여 컴포넌트(버튼, 텍스트 등) 명세서를 생성합니다.

### 🎨 5단계: DESIGN_SYSTEM.md (디자인 규칙)
- **목표:** 앱의 색상, 폰트 등 디자인 통일성을 위한 규칙을 정합니다.
- **실행 방법:** **[🚀 직접 시행]**을 눌러 프리미엄 느낌의 색상 조합과 폰트를 AI에게 추천받습니다.

### ✨ 6단계: AI UI 시안 생성
- **목표:** 앞서 정의한 모든 문서(PRD ~ 디자인 시스템)를 종합해 눈으로 볼 수 있는 디자인(HTML)을 만듭니다.
- **실행 방법:** **[🎨 AI 시안 자동 생성]** 또는 **[🚀 직접 시행]**을 누릅니다.
- **미리보기:** 결과가 나오면 **[👁️ 시안 미리보기]** 버튼을 눌러 디자인이 잘 나왔는지 확인해 보세요!

### ✅ 7단계: 개발자 승인 (로컬 폴더 세팅)
- **목표:** 코드를 개발하기 위한 실제 내 컴퓨터(로컬 PC)의 폴더 구조를 만듭니다.
- **실행 방법:** **[🚀 직접 시행]**을 누르면 AI가 `docs` 폴더에 문서들을 분류하고, `src` 폴더 등을 만드는 터미널 세팅 명령어(스크립트)를 줍니다. 
- **실습:** 결과물의 안내에 따라 실제 내 컴퓨터에 폴더를 만들고 파일을 정리해보세요.

### 💻 8단계: 코드 구현
- **목표:** 눈에만 보이던 시안에 생명(클릭, 저장 등 실제 기능)을 불어넣습니다.
- **실행 방법:** 지금까지 모인 모든 문서와 UI 시안 코드를 바탕으로 자바스크립트 로직을 추가해달라고 요청합니다. **[🚀 직접 시행]** 클릭!
- **미리보기:** **[👁️ 코드 구현 앱 미리보기]** 버튼을 눌러 버튼을 클릭하거나 글을 입력해보며 기능이 진짜 작동하는지 테스트해 봅니다.

### 🧪 9단계: 테스트
- **목표:** 앱이 고장날 수 있는 예외 상황(버그)을 미리 찾아냅니다.
- **실행 방법:** 구현된 코드에서 발생할 수 있는 잠재적 버그 3가지와 방어 방법을 AI에게 묻습니다.

### 🚀 10단계: 배포
- **목표:** 나 혼자 보던 앱을 인터넷(URL)에 올려 다른 사람들에게 공유합니다.
- **실행 방법:** 생성된 최종 코드를 Vercel이나 Netlify 같은 무료 사이트에 배포하는 방법을 AI에게 안내받아 따라 합니다.

### 💬 11단계: 피드백 반영
- **목표:** 친구나 사용자들의 의견을 듣고 다음 버전(v1.1)을 기획합니다.
- **실행 방법:** 프롬프트 입력창에 사용자들이 아쉬워했던 점을 적고 다음 업데이트 계획을 수립합니다.

## 💾 작업 결과물 일괄 다운로드하기

앱 개발의 모든 단계를 마쳤거나 중간에 내용을 저장하고 싶다면?

1. 우측 상단의 **[💾 전체 파일 다운로드]** 버튼을 클릭합니다.
2. 다운로드할 단계의 체크박스를 확인합니다. (기본적으로 모두 선택되어 있습니다.)
3. **[선택 항목 다운로드]** 버튼을 누릅니다.
4. `앱이름_1_idea.md`, `앱이름_8_Implementation.html` 처럼 각각의 단계에 맞게 정리된 문서와 소스코드 파일이 자동으로 내 컴퓨터에 저장됩니다!
