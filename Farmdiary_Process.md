# 🍎 과수원 일기 (Orchard Diary) - 11단계 바이브코딩 개발 일지

이 문서는 AI 앱 서비스 기획 프롬프트를 바탕으로 기획부터 개발, 배포까지 11단계를 거쳐 '과수원 일기' 앱을 만든 과정의 요약입니다.

---

## 💡 [1단계] 아이디어 도출
* **배경**: 2년차 귀농 50대 초보 농부로, 날씨에 따라 매일 무슨 일을 해야 할지 막막함.
* **해결책**: 매일 접속 시 큼직한 날씨 정보를 보여주고, 사과/배/복숭아/포도 등 주요 작물에 대한 '오늘의 필수 작업'을 추천 및 체크할 수 있게 하는 모바일 맞춤형 웹앱 기획.

## 📄 [2단계] PRD.md (요구사항 정의)
* **MVP 3가지**: 1) 직관적인 날씨 대시보드 2) 4가지 작물별 필수 작업 체크리스트 3) 메모와 함께 저장되는 로컬 영농 일지 기능 정의 완료. (상세 내역은 `Farmdiary/docs/01_PRD.md` 참고)

## 🔄 [3단계] USER_FLOW.md
* 앱 실행 ➡️ 메인 대시보드 진입 ➡️ 상단 날씨 확인 ➡️ 하단 작물 탭(사과 등) 터치 ➡️ 리스트 체크 ➡️ 하단 저장 버튼 클릭 순서로 매우 단순한 동선 기획.

## 🖥️ [4단계] UI_SPEC.md & 🎨 [5단계] DESIGN_SYSTEM.md
* **디자인 원칙**: 타겟층(50대)을 배려해 글씨 크기를 키우고(최소 16px, 보통 18px 이상), 탭과 버튼(min-h 48px) 터치 영역을 넉넉하게 잡음. 초록색(#16a34a)을 메인 색상으로 지정.

## ✨ [6단계] AI 시안 생성 & ✅ [7단계] 개발자 승인
* AI가 작성한 HTML 시안을 확인한 후, 로컬 PC에 `Farmdiary` 폴더와 `docs/` 폴더 구조를 자동 스크립트로 생성하여 명세서들을 저장 완료.

## 💻 [8단계] 코드 구현
* 단일 파일 `Farmdiary/index.html`에 Tailwind CSS 기반의 반응형 모바일 UI를 코딩.
* 바닐라 자바스크립트를 이용해 작물 탭 클릭 시 리스트 변경 로직과, 체크박스 및 메모를 기기의 `localStorage`에 자동 저장하는 로직 구현 완료.

## 🧪 [9단계] 테스트
* 탭 간 이동 시 기존 체크 데이터가 유지되는지 검증 (Pass). 
* 오늘 날짜 기준(`diary_YYYY_MM_DD`)으로 로컬 스토리지가 정상적으로 기록/로드되는지 검증 (Pass).

## 🚀 [10단계] 배포 (GitHub & Vercel)
* **GitHub**: 생성된 `Farmdiary` 소스코드와 기획 문서들을 기존 저장소(`vibe-coding-workflow`)에 Commit & Push 완료.
* **Vercel**: GitHub 저장소에 코드가 Push됨과 동시에 Vercel에 연동된 환경설정(`vercel.json`)을 통해 자동으로 글로벌 배포망에 릴리즈 됨.

## 💬 [11단계] 피드백 반영 계획
* **향후 계획(v1.1)**: 향후 농사 일정 알림 기능 추가, 그리고 사진(과수원 상태) 첨부 기능, 날씨 API 연동 등을 사용자 피드백으로 가정하고 다음 버전 백로그에 등록함.
