# 🎯 바이브코딩 (Vibe Coding) 앱 서비스 분석 및 자동화 전략 보고서

이 보고서는 '바이브코딩 11단계 워크플로우' 앱의 전체 코드를 분석하여, **에이전트 하네스(Agent Harness)** 기준에 따른 현재 상태 진단과 각 단계별 에이전트 활용 자동화 방안을 정리한 문서입니다.

---

## 1. 앱 전체 코딩 분석 요약

현재 앱은 **Vanilla JS + Tailwind CSS + OpenAI API**를 기반으로 한 인터랙티브 학습용 웹 애플리케이션입니다.

- **핵심 아키텍처**: '프롬프트 릴레이(Prompt Relay)' 시스템을 통해 이전 단계의 산출물을 다음 단계의 입력값으로 누적 전달하여 문맥(Context)을 유지합니다.
- **주요 로직**: `app.js`에서 11단계를 관리하며, `scenarios.js`의 템플릿을 기반으로 유저의 입력을 받아 AI(OpenAI)에게 전달하고 결과를 마크다운으로 렌더링합니다.
- **특이점**: 단순한 챗봇을 넘어 **6단계(UI 시안)**와 **8단계(코드 구현)**에서 실제 렌더링 가능한 코드를 생성하고 '미리보기'를 제공하는 실천형 프레임워크입니다.

---

## 2. 하네스(Harness) 기준 검토 및 진단

에이전트 하네스(Harness)란 AI 에이전트가 특정 목표를 달성할 수 있도록 설계된 **구조화된 환경**을 의미합니다. 아래 4가지 핵심 기준으로 현재 앱을 검토합니다.

| 기준 (Criteria) | 현재 상태 (Current) | 진단 및 개선 제안 |
| :--- | :--- | :--- |
| **Context Continuity** | 프롬프트 릴레이를 통해 이전 단계의 MD 내용을 누적하여 전달함 | **[우수]** 단계가 진행될수록 AI가 앱의 전체상을 더 명확히 이해함. |
| **Role-Specific Instructions** | 단계별로 기획자, 디자이너, 개발자 등 역할을 부여함 | **[보통]** 시스템 프롬프트(System Message)를 더 정교화하여 페르소나를 강화할 필요가 있음. |
| **Artifact generation** | MD 문서(PRD, Spec 등)와 HTML 소스 코드를 생성함 | **[우수]** 구조화된 문서화를 강제하여 개발 품질을 높임. |
| **Tool Use & Execution** | 생성된 코드를 `iframe`으로 미리보기만 수행함 | **[미흡]** 실제 파일 쓰기(File Write), 터미널 실행 등 도구 활용(Tool Use) 자동화가 부족함. |

---

## 3. 단계별 에이전트 활용 및 자동화 전략

현재는 사용자가 버튼을 눌러야 다음으로 넘어가지만, 고도화된 **에이전트 워크플로우**를 적용하면 다음과 같이 자동화할 수 있습니다.

### [Phase 1: 기획 및 설계 (1~5단계)]
- **활용 에이전트**: **Product Architect Agent**
- **자동화 방안**:
    - 1단계 아이디어 확정 시, 에이전트가 자동으로 2~5단계(PRD, User Flow, UI Spec, Design System)를 **일괄 생성(Chain of Thought)**.
    - 각 문서 간의 모순(Inconsistency)을 에이전트가 스스로 검토하여 보정.

### [Phase 2: 비주얼 및 구조화 (6~7단계)]
- **활용 에이전트**: **UI/UX & DevOps Agent**
- **자동화 방안**:
    - **6단계**: 단순 HTML 생성을 넘어 디자인 에이전트가 이미지 생성 툴을 사용하여 실제 고퀄리티 자산(Asset)을 생성.
    - **7단계**: (현재) 스크립트만 제공 -> (자동화) 에이전트가 직접 터미널 명령을 통해 로컬 프로젝트 폴더를 생성하고 초기화 수행.

### [Phase 3: 개발 및 검증 (8~9단계)]
- **활용 에이전트**: **Software Engineer & QA Agent**
- **자동화 방안**:
    - **8단계**: 에이전트가 코드를 파일로 직접 작성. 모듈별 분할 구현 자동화.
    - **9단계**: 에이전트가 브라우저 테스트 도구를 사용해 실제 앱을 구동하고, 버그 발견 시 **자가 수정(Self-Healing)** 루프 실행.

### [Phase 4: 배포 및 운영 (10~11단계)]
- **활용 에이전트**: **Cloud & Feedback Agent**
- **자동화 방안**:
    - **10단계**: CI/CD 도구를 활용해 에이전트가 즉시 배포를 실행하고 최종 URL을 보고.
    - **11단계**: 실제 사용자 리뷰나 로그를 에이전트가 분석하여 다음 버전 업데이트를 위한 `Backlog` 자동 생성.

---

## 4. 최종 자동화 로드맵 제안

1. **에이전트 협업 체계 구축**: 단일 AI와의 대화가 아닌, 각 단계의 전문가 에이전트들이 협업하는 'Multi-Agent' 시스템으로 전환.
2. **도구 활용(Tooling)의 실체화**: 단순히 텍스트를 생성하는 수준에서 벗어나, 에이전트가 파일 시스템, 브라우저, 터미널을 직접 제어하도록 설계.
3. **지속적 자동화(CA - Continuous Automation)**: 1단계 입력만으로 10단계 배포까지 '초안'이 완성되는 **One-Click MVP Pipeline** 구축 가능.
